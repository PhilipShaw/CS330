# CS330 README

## How I approach designing software and development, and how computer science helps me reach those goals

Software development has always felt most natural to me when I approach each design by initially creating a minimum viable product. This allows me to confirm that a concept behaves as expected and become aware of any unforeseen issues that need to be addressed early on. However, this project (and CS330 as a whole) has expanded my perspective on improving this type of workflow.

In previous courses, I was introduced to Agile methodologies (namely Scrum), which promote modular development. This allows teams to divide and conquer, simultaneously producing iterative work on multiple parts of a project. Although this made sense in theory, the concept seemed potentially unwieldy without a clear initial understanding of the broader project.

Working through CS330's main project in structured milestones helped me realize that even iterative workflows require a clear vision from the beginning in order to be successful. I used that approach in the final week to create the background building models and their textures. In the first iteration, I created the shapes of the building outlines with differentiating colors:

Afterward, I modularized and cleaned up certain aspects of the code.

I've always been under the impression that iterative development required an ability to be comfortable working with uncertainty and bringing parts together later on. When I began this course, I had a mindset that a project either needed to follow a strictly sequential or "waterfall" method, or be done essentially "flying blind" in order to maximize efficiency, hastily assembling the pieces afterward. That thinking has honestly made me lose some love for software development. The format of CS330 helped me realize that I don't have to abandon the creative process that works for me. Instead, iteration and structure can sharpen it in a way that becomes more efficient and cohesive with other developers.

One of the aspects I'm most proud of is gaining the fundamental skills to arrange models, source textures, edit texture images, and apply those textures. Because these are new concepts to me, it felt meaningful to apply them successfully. Additional proficiencies I've gained include a stronger applied knowledge of C++, as well as an introduction to graphics rendering pipelines and OpenGL variants. I was also introduced to supporting libraries such as GLEW and GLFW (for window creation and input handling).

Overall, this course has helped me better understand how to balance structure and flexibility, and has demystified iterative planning through application. While I now see computer science as more of a personal pursuit than a long-term career path, I plan to apply the skills I've developed in this course to future personal projects. One idea I've been particularly interested in is exploring gamified gym equipment, where interactive visuals could enhance the workout experience. The fundamental computational graphics and visualization concepts from CS330 will be very useful toward that goal.
